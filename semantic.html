Compile Structure

Unlike many CSS frameworks, Start is designed to work with theme files. As such, it uses its own theme file out-of-the-box.

File structure in a nutshell

lib/
theme.sass
style.sass

Here's the gist:

  1. Everything in lib shouldn't be edited.
  2. theme.sass defines your overall look and feel. Colors, typography, etc, but does not assign styles to the DOM.
  3. style.sass will assign styles to dom elements.

Browser Support

Which browsers does start.css support? Latest.

IE11+, Chrome, Firefox 16+, and Safari (sort of) are all auto updating browsers. Because of this shift in the browser support ecosystem, we state the current latest browsers on the day of a Start.css stable release are the browsers that version supports.

For older browser support, download an earlier version of Start.css.

End Concatenation

This is a small trick to prevent the last line of a paragraph from containing only one word.

So instead of:

lorem ipsum dolor sit
amet

End concatenation produces:

lorem ipsum dolor
sit amet

Using the jQuery plugin is simple:

$(element).s_endconcat()

Where element is the element to have enconcatenation applied.

Grid

Start uses a 12 column grid. To create a gridded row, start with a container inheriting the row() styling.

<main></main>
 
main { @include row(); }

Columns are defined with the col() mixin and cols(x) measuring styles where "x" is a number between 1-12.

Using cols(x) without a measuring class will default to cols(12)

.sidebar
.content

<main>
  <div class="sidebar">.sidebar</div>
  <div class="content">.content</div>
</main>
 
main { @include row(); }
.sidebar { 
  @include col();
  @include cols(4);
}
.content {
  @include col();
  @include cols(8);
}

Responsive Grid

Columns will target all screen sizes by default. To target one screen size, add the min-width of that screen as a second parameter of cols(x, y), where y is the width in pixels.

col() + cols(4, 768px)Full width on mobile, 1/3 on screens >768px.
col() + cols(8, 768px)Full width on mobile, 2/3 on screens >768px.
<main>
  <div class="sidebar">Full width on mobile, 1/3 on screens >768px.</div>
  <div class="content">Full width on mobile, 2/3 on screens >768px.</div>
</main>
 
main { @include row(); }
.sidebar { 
  @include col();
  @include cols(4, 768px);
}
.content { 
  @include col();
  @include cols(8, 768px);
}

Container

To contain the page in a nice, centered wrapper, use wrapper()

By default, this will have no effect for pages under 1200px. Screens larger than this will keep your site in a 1200px wrapper with equal spacing on the left and right.

For different sizes, add an additional class with the form s-[size] where [size] is xs, sm, md, or lg.

<body>
  <main>
    <aside>
      <h5>Sidebar</h5>
    </aside>
    <section>
      <h1>Main section</h1>
    </section>
  </main>
</body>
body    { @include wrapper(); }
main    { @include row(); }
aside   { 
  @include col();
  @include cols(4);
}
section { 
  @include col();
  @include cols(8);
}

Growls

Growls are unobtrusive pop up messages called via JavaScript. A basic call might look like this:

$.sGrowl({
   icon: 'save',
   title: 'Saved',
   text: 'Your document has been saved'  
 })

Because it's never necessary to manually create the growl markup, growls are automatically created with the class .s-growl.

Parameters

Parameter Default Description
icon — Icon to appear before the heading. Choose from the Font Awesome 4 library without the preceding "fa-"
title — Bold-faced title to appear on the first line.
text — Main text under the title. HTML allowed.
delay (Infinite) Delay in miliseconds before dismissing growl. Ignore this parameter to keep growl permanent.
container #s-growls HTML element where growl should be sent. Write in CSS notation.
color green Color of growl (red/orange/yellow/green/blue/violet).
class — Additional classes to apply to growl. Separate multiple classes with a space.

Advanced

Here is a growl that will appear in red in the container below and disappear after 3 seconds.

Growl will appear above.

$.sGrowl({
  color: 'red', 
  icon: 'book',
  title: 'Test',
  text: 'Testing...1, 2, 3', 
  delay: 3000, 
  container: '#example-container'
})

Effects

Fixed Pane

The fixed pane is a full window wrapper to affix elements to the screen while still obeying the grid.

Effectively, this creates a full fixed pane over the entire screen in which elements can be arranged as they are anywhere else on your page.

For example, see the sidebar on this page.

Say you had this layout, where a sidebar appears on 1/3 of the screen:

<body>
  <main>
    <aside>Sidebar</aside>
    </section>Main</section>
  </main>
</body>
 
body { @include wrapper(); }
main { @include row(); }
aside {
  @include col();
  @include cols(4);
}
section {
  @include col();
  @include cols(8);
}

When you scroll down on the page, the sidebar will scroll with it. Here's how we can pop it out:

<body>
  <div class="fixed">
    <div class="body">
      <main>
        <aside>Sidebar</aside>
      </main>
    </div>
  </div>
  <main>
    </section>Main</section>
  </main>
</body>
 
body, .body { @include wrapper(); }
main        { @include row(); }
aside {
  @include col();
  @include cols(4);
}
section {
  @include col();
  @include cols(8);
  @include bump(4);
}
.fixed { @include fixed-pane(); }

Playing nicely with mouse events

The full screen nature of the fixed pane can cause annoyances, such as causing every element selection from Chrome's web inspector to select the whole page instead of the element being targeted.

To fix this, use the jQuery plugin $.sFixedPane().

$('.fixed').sFixedPane()

This will allow elements in the pane to scroll when overflowing the viewport but also to "hide" click interactions when interacting outside the fixed pane's elements.
